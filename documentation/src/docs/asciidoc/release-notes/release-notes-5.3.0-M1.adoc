[[release-notes-5.3.0-M1-junit-platform]]
=== JUnit Platform

==== Bug Fixes

* Full stacktrace is printed to the console, when running the `ConsoleLauncher`
  in `--details verbose` mode.

==== Deprecations and Breaking Changes

* ❓

==== New Features and Improvements

* Added `--fail-if-no-tests` command line option to `ConsoleLauncher`. When this option is enabled
and no tests are found for execution the launcher will fail and exit with the non zero code `2`.


[[release-notes-5.3.0-M1-junit-jupiter]]
=== JUnit Jupiter

==== Bug Fixes

* ❓

==== Deprecations and Breaking Changes

* ❓

==== New Features and Improvements

* New `assertThrows` methods in `Assertions` provide a more specific failure
  message if the lambda returns a result instead of throwing the expected
  exception.
* New `get<Class>(index)` Kotlin extension method to make `ArgumentsAccessor` friendlier
  to use from Kotlin.
